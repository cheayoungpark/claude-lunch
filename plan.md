# 오늘 뭐 먹지? — 기능 로드맵

## 프로젝트 개요

점심 메뉴를 고르는 데 드는 시간과 고민을 없애주는 웹앱.
빌드 도구 없이 순수 HTML/CSS/JS로 만들어졌으며, 별도 서버 없이 브라우저만으로 동작한다.
기록은 브라우저의 localStorage에 저장되어 새로고침 후에도 유지된다.

- **배포**: GitHub Pages — https://cheayoungpark.github.io/claude-lunch/
- **기술 스택**: HTML5 / CSS3 / Vanilla JS / localStorage
- **대상 사용자**: 매일 점심 메뉴를 고민하는 직장인

---

## 핵심 기능

| 기능 | 설명 |
|------|------|
| 랜덤 추천 | 카테고리(한식/중식/일식/양식/분식) 필터 + 플래시 애니메이션으로 메뉴 추천 |
| 제외 설정 | 어제 저녁에 먹은 메뉴를 선택하면 추천에서 자동 제외 |
| 점심 기록 | 추천 결과를 버튼 하나로 오늘 기록에 저장, 최근 31일 보관 |
| 기록 수정/삭제 | 목록·달력 뷰 양쪽에서 잘못된 기록을 수정하거나 삭제 |
| 달력 뷰 | 월별 달력에서 날짜마다 이모지로 먹은 메뉴를 한눈에 확인 |
| 목록 뷰 | 최근 기록을 날짜 역순으로 나열, 각 항목 수정·삭제 가능 |

## 프로젝트 구조

```
claude_lunch/
├── index.html     # 레이아웃 — 좌(메뉴 추천) / 우(기록) 2단 가로 구성
├── style.css      # 전체 스타일 (오렌지 그라디언트 테마, 반응형)
├── app.js         # 모든 로직
│   ├── menus[]        데이터: 30개 메뉴 (이름, 이모지, 카테고리, 설명)
│   ├── recommend()    추천 로직 + 플래시 애니메이션
│   ├── recordLunch()  기록 저장 (localStorage)
│   ├── editRecord()   목록 뷰 수정
│   ├── calEditRecord() 달력 뷰 수정
│   ├── renderHistory() 목록 뷰 렌더링
│   └── renderCalendar() 달력 뷰 렌더링
├── CLAUDE.md      # Claude Code 가이드
└── plan.md        # 이 파일
```

**localStorage 데이터 구조**
```json
lunchRecords: [
  { "date": "2026-03-20", "name": "비빔밥", "emoji": "🍚", "category": "한식" },
  ...
]
```

## 사용자 경험 시나리오

### 시나리오 1 — 매일 아침 점심 고민 해결
> "오늘 뭐 먹지?" 버튼 하나로 30초 안에 결정

1. 사이트 접속
2. 어제 저녁에 먹은 메뉴 선택 (제외)
3. 카테고리 선택 (오늘은 한식 말고 다른 거)
4. 추천 버튼 클릭 → 결과 확인
5. 마음에 안 들면 다시 뽑기
6. 결정 후 "오늘 점심으로 먹었어요" 클릭 → 자동 기록

### 시나리오 2 — 지난 한 달 식습관 돌아보기
> 달력 탭에서 한눈에 확인

1. 달력 탭 전환
2. 월별로 어떤 메뉴를 먹었는지 이모지로 시각적 확인
3. 특정 날짜 클릭 → 메뉴 상세 확인
4. 잘못 기록된 날짜 → 수정 버튼으로 바로 수정

### 시나리오 3 — 다른 메뉴를 먹었을 때
> 추천받은 메뉴가 아닌 다른 걸 먹었어도 기록 가능 (미구현)

1. 추천은 참고만 하고 실제로 다른 메뉴를 먹음
2. 기록 패널에서 해당 날짜의 ✏️ 버튼 클릭
3. 드롭다운에서 실제 먹은 메뉴로 변경
4. 변경 완료

### 시나리오 4 — 점심 메뉴 반복 방지 (미구현)
> 이번 주 내내 같은 메뉴가 추천되는 상황 방지

1. 최근 기록을 바탕으로 자주 먹은 메뉴 자동 감지
2. 추천 시 최근 N일 내 먹은 메뉴는 제외
3. 설정에서 제외 기간 조정 (3일 / 7일 / 14일)

## 개선이 필요한 UX 포인트

- 모바일에서 2단 레이아웃이 좁아짐 → 세로 전환 분기점 설정 필요
- 기록이 많아질수록 목록 스크롤이 길어짐 → 날짜 검색 또는 필터 필요
- 추천 결과가 이전과 같을 때 사용자가 인식하기 어려움 → "방금 전과 다른 메뉴" 강조 표시
